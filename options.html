Y/TXT Options

Configure providers, model/tool definitions, and the optional Keystone companion for safer local key storage.

Getting Started

Y/TXT works in two modes.

Fastest path: paste at least one provider key below, set the correct default provider, save, and start using the extension immediately.

Safer path: install Keystone and store provider keys outside the browser, then let Y/TXT tunnel requests through the local host.

You do not need Keystone just to get started. Keystone is an optional upgrade for stronger local secret handling.

⚠️ Security Warning

Preferred: store provider keys in Keystone and let Y/TXT tunnel requests through the local host. Local browser storage is supported as a fallback when you want the fastest setup.

If local browser storage is active, use low-limit keys and treat this browser profile as a trusted environment.

Provider Keys

Each provider entry comes from Providers Config below. Keystone can only tunnel providers it knows; unknown provider IDs fall back to local extension storage only.

Quick setup: enter one key, click Save Settings, and make sure Default Provider below matches the provider you actually configured.

Default Provider

Used only when a tool step does not set its own provider. Tools can still choose a provider explicitly.

Tools Config

Define and customize tools as JSON. Each category has a label and a tools array. Supports {{input}}, {{previous}}, {{originalInput}}, and option-id placeholders in prompts. Use a steps array to build chained tools.
Changes take effect when the popup is re-opened.

Providers Config

Define provider IDs, labels, adapter type, and upstream base URLs here. Tools and model sets reference these provider IDs.

Models Config

Map tier names (e.g. "coding", "creative") to provider-specific models. A string value uses the tool's temperature; an object with "temperature": null suppresses it.
Invalid provider mappings are rejected instead of silently falling back.
Changes take effect immediately.

Keystone

Test the local Native Messaging bridge and open the local admin UI. Keystone is optional, but preferred once you want secrets stored outside the browser.

General Info

Why this extension exists

Y/TXT treats plain-language instructions as configurable tools. Instead of building a separate app for every narrow text workflow, you can often describe the job in human language and let a model handle it.

That means custom tools, custom providers, and model mappings can all be composed here without changing extension code.

API Keys

LLM providers require API keys so requests can be authenticated and billed. Chrome extensions can store data, but they do not offer a truly secure secret vault for extension-managed API keys.

If secure API-key handling matters, use Keystone. It stores secrets outside the extension and lets Y/TXT tunnel requests through a local trusted host instead of keeping the key in browser storage.
